مفهوم الفئات الزائفة

**الفئات الزائفة (Pseudo-classes)** هي كلمات مفتاحية تُضاف إلى مُحدِّدات CSS لتحديد حالة خاصة للعنصر الذي نريد تنسيقه. على سبيل المثال، يمكن استخدامها لتحديد شكل الزر عند مرور الماوس فوقه، أو لتنسيق الرابط بعد زيارته.

يتم كتابة الفئة الزائفة بعد اسم المحدد، مسبوقة بنقطتين رأسيتين (:).

حالات الروابط

تُستخدم الفئات الزائفة بشكل شائع لتنسيق الروابط بناءً على حالتها:

  • :link: يُطبَّق على الروابط التي لم يتم النقر عليها بعد.
  • :visited: يُطبَّق على الروابط التي تمت زيارتها بالفعل.
  • :hover: يُطبَّق عند مرور مؤشر الماوس فوق الرابط.
  • :active: يُطبَّق عند النقر على الرابط (في لحظة الضغط).

مثال: تنسيق حالات الرابط

a:link {
    color: blue;
}
a:visited {
    color: purple;
}
a:hover {
    color: red;
}
a:active {
    color: green;
}

تفاعلات المستخدم الشائعة

يمكنك استخدام الفئات الزائفة لتنسيق أي عنصر عند تفاعل المستخدم معه، وليس الروابط فقط:

:hover

يُطبق هذا على أي عنصر عندما يمر الماوس فوقه. يمكن استخدامه لإضافة تأثيرات بصرية جذابة على الأزرار، الصور، أو البطاقات.

مثال: تأثير على زر

button:hover {
    background-color: #3498db;
    transform: scale(1.1);
}

:active

يُطبق هذا على العنصر في اللحظة التي يتم فيها النقر عليه.

مثال: تأثير على زر عند النقر

button:active {
    background-color: #2980b9;
    transform: scale(0.95);
}

:focus

يُطبق هذا على العنصر الذي يتم تحديد موقعه حاليًا (مثل حقل إدخال عند النقر عليه أو التنقل إليه باستخدام لوحة المفاتيح).

مثال: تنسيق حقل الإدخال عند التركيز عليه

input:focus {
    border-color: #3498db;
    box-shadow: 0 0 5px rgba(52, 152, 219, 0.5);
}

الفئات الزائفة لعناصر النماذج

هناك فئات زائفة خاصة تستخدم لتنسيق عناصر النماذج بناءً على حالتها:

  • :valid: يُطبق عندما تكون قيمة حقل الإدخال صحيحة.
  • :invalid: يُطبق عندما تكون قيمة حقل الإدخال غير صحيحة.
  • :required: يُطبق عندما يكون حقل الإدخال إلزامياً.
  • :disabled: يُطبق عندما يكون العنصر مُعطَّلاً.
